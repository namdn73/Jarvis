# ImplementationPlan

This folder contains the implementation plans for Jarvis. Read the relevant plan
before starting any coding session — each plan file contains module-by-module
instructions, confirmed design decisions, and code patterns to follow.

## Files

| File | What it covers |
|---|---|
| [general/architecture.md](general/architecture.md) | Shared contract: state machine, data flow, WebSocket protocol, REST API, SQLite schema. **Read this first.** |
| [general/task.md](general/task.md) | Master task list — 11 sessions with checkboxes. Track progress here. |
| [backend/plan.md](backend/plan.md) | Full backend implementation: every module, its logic, and code patterns. |
| [frontend/plan.md](frontend/plan.md) | Full frontend implementation: components, hooks, state, and styling. |

## How to use

Each session in `task.md` is designed for a separate chat session with Claude.
Start a new chat, reference the relevant plan files, and work through the
session's checklist. Tick boxes as tasks complete.
